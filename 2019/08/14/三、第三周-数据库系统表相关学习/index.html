如何利用数据库的功能读写文件

读取文件

MySQL提供了允许load data infile命令高速地从一个文本文件中读取行，并装入一个表中，看下此命令的说明：

load data infile '文件名' into table 表名

满足三个条件;

  • 1.文件权限：chmod o+r pathfile（由于mysql是其他者用户）
  • 2.文件大小：max_allowed_packet必须小于最大包大小（使用select
    @@max_allowed_packet;可查看）
  • 3.允许读写的目录：secure_file_priv，如果为NULL则不允许导入或导出，如果为/tmp则只
    能在tmp目录下，如果没有值则不限制

查看配置
mysql> show global variables like '%secure%';

windows下

  • 修改mysql.ini文件，在[mysqld]下添加条目：secure_file_priv=’’

linux下

  • 在/etc/my.conf或者/etc/my.cnf的[mysqld]下面添加secure_file_priv=’’选项

写入文件

MySQL提供了into outfile函数，可以将一条select语句的结果写到MySQL进程所有者拥有的完全可读可写的文件中。

需要条件

  • 1.文件的写入权限：chmod o+x
  • 2.secure_file_priv读写目录

最好是txt文件，否则会乱码

更改查询自动写入文件，使用pager[cmd]更改mysql的查询输出，cmd为linux标准命令
将所有查询都输出到这个文件

学习数据库系统表的功能，利用sql来查询库名、表名、内容以及当前用户等基本信息

information_schema
首先这个数据库是mysql自带，它提供了访问数据库的元数据的方式
元数据就是关于数据的数据，可以说成是数据词典和系统目录
信息数据库，其中保存着关于mysql服务器所维护的所有其他数据库的信息，如数据库名，
数据库表，表栏的数据类型与访问权限，在information_schema中，有数个只读表，实际
上是视图，而不是基本表，所有你无法看到与之相关的任何文件

获取库名
SELECT schema_name from information_schema.schemata;

获取表名
SELECT table_name from information_schema.tables WHERE table_schema='sys';


获取student表的字段名 (列)
SELECT column_name from information_schema.columns WHERE table_schema='stx' and table_name='student';

通过mysql库下的user爆用户名和HASH(mysql8)
select Host,user,authentication_string FROM mysql.USER

一些其他操作

  • 获取数据库版本 stlect version()
  • 获取当前操作系统select @@version_compile_os
  • 获取当前用户：select user();
  • 文件保存路径select @@datadir;
  • 数据库开放端口：select @@port;
  • 获取主机名称：select @@hostname
