基于时间的盲注

函数 SLEEP（duration）

函数使用说明：睡眠（暂停）时间为duration参数给定的秒数，然后返回0.若SLEEP被终端，它会返回1.

Select * from table where id = 1 and sleep(4);

可以看到执行sleep函数后，运行时间变成3秒

其实基于时间的注入和我们最早的

做出判断的行为是一种类似的思路，为什么and1=1,1=2可以帮助我们判断？这里存在了一个逻辑，这个逻辑会影响到查询结果，说明我们插入的东西被当成代码去执行了。这里也是一样的添加sleep后出现结果的时间变慢了，就可以判断sleep产生了效果，我们注入的代码和正常访问产生了差异。


可以看到图片2，表中有3个数据，，输入and逻辑语句，会有延迟，可以理解，一般来说and逻辑前后都为真才会返回结果

输入or逻辑，发现延迟6秒左右，这是因为，整张表有3条数据，id=2逻辑为真，后面的sleep函数就不会执行，继续会判断下一个，id=3，为假，执行sleep函数，依次下去，因此会有6秒左右的延迟。
